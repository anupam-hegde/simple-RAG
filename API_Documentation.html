RAG Document Assistant

REST API Reference & Architecture Documentation

1. System Architecture

The RAG Document Assistant is built on a modular architecture using FastAPI for the backend, Streamlit for the frontend, ChromaDB for vector storage, HuggingFace for local embeddings, and Groq (Llama 3.3) for lightning-fast LLM generation.

flowchart TD User([User]) -->|Uploads PDF / Asks Question| UI[Streamlit Frontend] subgraph Backend [FastAPI Backend] API[API Router] RAG[RAG Service] CH[Chat History Service] API <--> RAG API <--> CH end UI <-->|REST / SSE Streams| API subgraph Data Layer Chroma[(ChromaDB)] Storage[File System JSON] end subgraph External Models HF[HuggingFace Local Embeddings] Groq[Groq API - Llama 3.3] end RAG <-->|Fetch Context| Chroma RAG -->|Generate Embeddings| HF RAG <-->|Generate Answer| Groq CH <-->|Persist Sessions| Storage

2. Authentication

The API requires authentication via an HTTP header. Set your desired API Key in the backend .env file under API_KEY. If left empty, authentication is disabled (dev mode). 

Header: X-API-Key: <your_secret_key>

3. Document Endpoints

POST /api/upload

Upload a document (PDF, TXT, MD). The file is processed synchronously: text is extracted, chunked, embedded locally, and stored in ChromaDB.

Request Form-Data

FieldTypeRequiredDescription
fileFileYesThe document to ingest.

Response (200 OK)

{
  "status": "success",
  "filename": "Project_Orion_Specs.pdf",
  "message": "File 'Project_Orion_Specs.pdf' processed successfully. 14 chunks indexed."
}
POST /api/upload/progress

Upload a document and receive real-time Server-Sent Events (SSE) detailing the progress of text extraction, chunking, and embedding. Ideal for UI progress bars.

Response Stream (text/event-stream)

data: {"stage": "saving", "message": "Saving uploaded file…", "pct": 5}

data: {"stage": "extracting", "message": "Extracting text from document…", "pct": 15}

data: {"stage": "chunking", "message": "Text split into 8 chunks...", "pct": 30, "total": 8}

data: {"stage": "embedding", "message": "Embedding chunk 1 of 8", "pct": 38, "current": 1, "total": 8}

data: {"stage": "done", "message": "Indexed successfully!", "pct": 100, "chunks": 8}

4. Chat & Generation Endpoints

POST /api/chat

Submit a question against the ingested documents. Returns a synchronous JSON payload with the LLM answer and the specific source chunks cited.

Request Body (application/json)

FieldTypeRequiredDescription
questionstringYesThe user query.
session_idstringNoChat session UUID for history.

Response (200 OK)

{
  "answer": "The Orion phone features 48 hours of battery life and recharges in 8 minutes. [Source: specs.pdf, Page: 1]",
  "sources": [
    {
      "text": "Carbon-lattice energy cell. 48 hours continuous usage, recharges in 8 minutes.",
      "filename": "specs.pdf",
      "page": 1
    }
  ],
  "session_id": "abc-123-def"
}
POST /api/chat/stream

Submit a question and stream the LLM response token-by-token via SSE. At the end of the stream, a dedicated sources event provides the citations.

Response Stream (text/event-stream)

data: The Orion phone
data:  features a battery
data:  life of 48 hours.

event: sources
data: [{"text": "...", "filename": "specs.pdf", "page": 1}]

event: done
data: [DONE]

5. Chat History Endpoints

GET /api/chat/history

Retrieve a list of all chat sessions (newest first).

[
  {
    "id": "abc-123",
    "title": "Battery life inquiry",
    "created_at": "2026-06-29T10:00:00Z"
  }
]
GET /api/chat/history/{session_id}

Retrieve the full conversation history (all user and assistant messages) for a specific session.

{
  "id": "abc-123",
  "title": "Battery life inquiry",
  "created_at": "2026-06-29T10:00:00Z",
  "messages": [
    {
      "role": "user",
      "content": "What is the battery life?",
      "sources": []
    },
    {
      "role": "assistant",
      "content": "48 hours.",
      "sources": [{"text": "...", "filename": "specs.pdf", "page": 1}]
    }
  ]
}
DELETE /api/chat/history/{session_id}

Permanently delete a chat session and all its messages.

{"status": "deleted", "session_id": "abc-123"}
